# Hospital Management Database (SQL)

## Project Overview
This project represents a Hospital Management Database designed to store and analyze patient, doctor, and department information.  
It helps in understanding patient distribution, doctor workload, and department-wise performance using SQL queries.

---

## Database Entities
- **Patient** – Stores patient details and assigned doctor
- **Doctor** – Stores doctor information and specialization
- **Department** – Stores hospital department details

---

## Key Business Questions Answered
- List of patients with their assigned doctors
- Patients categorized by department
- Number of patients treated by each doctor
- Department-wise patient count
- Identification of departments with high patient load
- Doctors handling more patients than average

---

## SQL Concepts Used
- Database schema design
- Primary and foreign keys
- Joins (INNER JOIN)
- Aggregate functions (COUNT, AVG)
- GROUP BY and HAVING clauses
- Subqueries

---

## Files Included
- `schema.sql` – Database structure (tables and relationships)
- `queries.sql` – SQL queries for data analysis
